# ZloVPO Server

**Учебный проект по дисциплине "Защита информации от вредоносного программного обеспечения"**

Сервер с JWT аутентификацией и ролевой авторизацией.

В файле `ZIoVPO_Server.postman_collection.json` расположена готовая коллекция запросов Postman для тестирования API.

---
## Технологии и требования

- Среда разработки **IntelliJ IDEA**
- **Java**
- **Spring Boot** 3.5.5+
- **Maven**
- **HTTP-запросы**: Postman
- **База данных**: PostgreSQL
- **Объектное хранилище файлов**: MinIO
- **Управление сертификатами и ключами ЭЦП**: keytool

## Основные эндпоинты

- Аутентификация 
  - `POST /api/auth/login` – вход (access + refresh токены)
  - `POST /api/auth/refresh` – обновление access токена
  - `POST /api/register` – регистрация
- Подсистемы лицензирования
  - `POST /license/create` – создание лицензии (администратор)
  - `POST /license/activate` – активация лицензии (пользователь)
  - `POST /license/renew` – продление лицензии (пользователь)
  - `POST /license/check` – проверка лицензии (пользователь)
- Управление антивирусными сигнатурами
  - REST API
    - `GET /signature/getAll` — Получить все актуальные сигнатуры
    - `GET /signature/getIncrement/{since}` — Получить инкремент изменений, произошедших после указанного момента времени (включая логически удалённые)
    - `GET /signature/getId` — Получить сигнатуры по списку идентификаторов (UUID передаются в теле запроса)
    - `POST /signature/create` — Создать новую сигнатуру (только для администратора)
    - `DELETE /signature/delete/{id}` — Логически удалить сигнатуру (только для администратора)
    - `PATCH /signature/update/{id}` — Обновить существующую сигнатуру (только для администратора)
    - `GET /signature/getHistory/{id}` — Получить историю изменений сигнатуры (только для администратора)
    - `GET /signature/getAudit/{id}` — Получить аудит действий над сигнатурой (только для администратора)
  - Бинарный API
    - `GET /signature/binary/getAll` — Получить все актуальные сигнатуры
    - `GET /signature/binary/getIncrement/{since}` — Получить инкремент изменений, произошедших после указанного момента времени (включая логически удалённые)
    - `GET /signature/binary/getId` — Получить сигнатуры по списку идентификаторов (UUID передаются в теле запроса)
  - Загрузка сигнатур на основе файлов API 
    - `POST /signature/files/upload` — Загрузка файла сигнатуры (multipart/form-data) с автоматическим анализом его содержимого, созданием записи о сигнатуре в БД и сохранением файла в приватном бакете MinIO. Доступ только для администратора.
    - `POST /signature/files/getUrls` — Получение словаря (UUID → pre‑signed URL) для списка идентификаторов сигнатур. Доступ только для администратора.
