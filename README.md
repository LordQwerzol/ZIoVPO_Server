# ZloVPO Server

**Учебный проект по дисциплине "Защита информации от вредоносного программного обеспечения"**

Сервер с JWT аутентификацией и ролевой авторизацией.

---
## Технологии и требования

- Среда разработки **IntelliJ IDEA**
- **Java**
- **Spring Boot** 3.5.5+
- **Maven**
- **HTTP-запросы**: Postman
- **База данных**: PostgreSQL

---
##  **Функциональность**
### Задание 1
- Аутентификация по JWT (access и refresh токены).
- Регистрация новых пользователей.
- Авторизация на основе ролей (USER, ADMIN).
- Хранение refresh токенов в БД (сущность UserSession).
- Защита REST эндпоинтов с проверкой токена и прав доступа.
- Поддержка HTTPS.
- Подключение к PostgreSQL через Spring Data JPA.

### Задание 2
- Реализация структуру таблиц и связей в PostgreSQL по ER-диаграмме
- Реализация операции создания лицензии 
- Реализация операции активации лицензии
- Реализация операции продления лицензии
- Реализация операции проверки лицензии
- Реализация класса `Ticket` для информирования пользователя 

### Задание 3
- Создать хранилище с приватным ключом и публичным сертификатом для создания и проверки подписи.
- Добавить значение публичного ключа в `Github variables`.
- Реализовать компоненты модуля ЭЦП, согласно требованиям.
- Подключить модуль ЭЦП к лицензии для подписания ответов. (Добавление `TicketResponse`).
- Убедиться, что подпись тикета формируется корректно.

## Основные эндпоинты

- Аутентификация 
  - `POST /api/auth/login` – вход (access + refresh токены)
  - `POST /api/auth/refresh` – обновление access токена
  - `POST /api/register` – регистрация
- Подсистемы лицензирования
  - `POST /license/create` – создание лицензии (администратор)
  - `POST /license/activate` – активация лицензии (пользователь)
  - `POST /license/renew` – продление лицензии (пользователь)
  - `POST /license/check` – проверка лицензии (пользователь)
